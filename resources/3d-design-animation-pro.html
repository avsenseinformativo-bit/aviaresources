Chapter 1: The Three.js Revolution

WebGL is hard. It requires writing shaders in GLSL, managing buffers, and understanding linear algebra. Three.js is the library that democratized 3D. It abstracts the complexity of WebGL into a friendly JavaScript API.

// The "Hello World" of 3D const scene = new THREE.Scene(); const camera = new THREE.PerspectiveCamera(75, window.innerWidth / window.innerHeight, 0.1, 1000); const renderer = new THREE.WebGLRenderer(); const geometry = new THREE.BoxGeometry(); const material = new THREE.MeshBasicMaterial({ color: 0x00ff00 }); const cube = new THREE.Mesh(geometry, material); scene.add(cube); camera.position.z = 5; function animate() { requestAnimationFrame(animate); cube.rotation.x += 0.01; cube.rotation.y += 0.01; renderer.render(scene, camera); } animate();

With just 15 lines of code, you have a 3D engine running in the browser. No plugins. No downloads.

Chapter 2: The Blender to Web Workflow

You don't code 3D models; you model them. Blender is the industry standard (and it's free). The secret to getting Blender models onto the web is the glTF format (GL Transmission Format).

Think of glTF as the "JPEG of 3D." It compresses geometry and textures into a tiny file. Workflow:

  1. Model high-poly mesh in Blender.
  2. Retopologize to low-poly (crucial for web performance).
  3. Bake lighting and normal maps.
  4. Export as `.glb` (binary glTF).
  5. Load in Three.js with `GLTFLoader`.

Chapter 3: Lighting and Materials (PBR)

Why do old 3D games look like plastic? Because they used simple lighting models (Phong/Lambert). Modern 3D uses PBR (Physically Based Rendering).

The 2 Key Properties of PBR:
1. Metalness: Is the surface a metal or a dielectric (plastic/wood)?
2. Roughness: How microscopic is the surface texture? (Mirror = 0, Matte = 1).

By tweaking these two values, you can simulate gold, rubber, wood, or skin.

Chapter 4: Performance Optimization

The web is not a PlayStation 5. You have limited resources. If your site runs at 15 FPS, users will leave. Optimization Checklist:

  • Draw Calls: Merge geometries. 1000 objects = 1000 draw calls = slow. 1 merged object = 1 draw call = fast.
  • Texture Size: Do not use 4K textures. 1024x1024 is usually enough.
  • Shadows: Real-time shadows are expensive. Bake them into textures whenever possible.

Chapter 5: Post-Processing (The "Juice")

Raw 3D renders often look dry. Post-processing adds the cinematic feel.

  • Bloom: Makes bright lights glow (Neon effect).
  • Depth of Field: Blurs the background to focus on the subject.
  • Ambient Occlusion (SSAO): Adds shadows in corners to add depth.

Chapter 6: React Three Fiber (R3F)

If you use React, do not use vanilla Three.js. Use React Three Fiber. It allows you to write 3D scenes as declarative components.

// The React Way <Canvas> <ambientLight /> <pointLight position={[10, 10, 10]} /> <Box position={[-1.2, 0, 0]} /> <Box position={[1.2, 0, 0]} /> </Canvas>

It manages the render loop, resize events, and cleanup automatically. It is the future of 3D web development.
